# TeslaNetEaseMusic

[简体中文](README.md) · [English](README.en.md) · **日本語**

[![License: AGPL-3.0](https://img.shields.io/badge/License-AGPL--3.0-blue.svg)](LICENSE)

海外で網易雲音楽(NetEase Cloud Music)を利用できない Tesla オーナー向けの、セルフホスト型オンラインプレーヤーです。各ユーザーが自分のインスタンスをデプロイし、自身のアカウントでログインして、車載ブラウザで楽曲を再生し、走行中もスクロール歌詞を表示します。

## 概要

- **セルフホスト** — 各ユーザーが独立したインスタンスを運用し、データは共有されません。
- **認証情報の分離** — ログイン Cookie はサーバー側のボリュームにのみ保存され、フロントエンドや車載ブラウザは保持しません。
- **車載向け最適化** — 大きな操作ボタン、暗く高コントラストな UI、カラオケ風スクロール歌詞。中央配置と左右分割の 2 レイアウトに対応。
- **公開アクセス** — Cloudflare Tunnel を同梱し、グローバル IP なしで HTTPS の URL を取得できます。

## 機能

- デイリーおすすめ、自分のプレイリスト、検索、パーソナル FM(パーソナライズド・ラジオ)
- 再生操作:再生/一時停止、前へ/次へ、シャッフル、リピート、シーク、音量
- カラオケ風スクロール歌詞(翻訳対訳対応)。インストゥルメンタルはプレースホルダーを表示
- 5 種類のテーマ、アルバム/アーティストページ、再生キュー、お気に入り
- 前回のキュー・再生位置・音量・設定を記憶
- Media Session 連携により、車載システムのメディアカードにジャケットと曲情報を表示

## アーキテクチャ

- **フロントエンド** — Vite + React + TypeScript。静的アセットとしてビルド。
- **BFF** — Node/Express サービス。QR ログイン、Cookie のサーバー側保存、リクエスト転送時のセッション注入を担当。
- **音楽 API** — `ncm-api` コンテナが提供する網易雲 API。
- **オーケストレーション** — Docker Compose が各サービスを起動。任意で Cloudflare Tunnel が HTTPS URL を公開。

### クイックスタート

Docker を導入済みの Mac で実行します。

```bash
git clone https://github.com/JovanCai/TeslaNetEaseMusic.git
cd TeslaNetEaseMusic
./deploy.sh
```

スクリプトはサービスを起動し、スマホでの QR ログインを案内し、`https://…` の公開 URL を出力します。その URL を Tesla のブラウザで開くと再生できます。

## 設定

設定はプロジェクト直下の `.env` ファイルで指定します。変更後は再デプロイが必要です。

| 変数 | 既定値 | 説明 |
|---|---|---|
| `APP_PORT` | `80` | 公開サービスのポート。使用中の場合は空いている別ポートに変更 |
| `REGION_UNLOCK` | `false` | リージョン解除。有効時、バックエンドがランダムな中国 IP でリクエストし、海外でグレーになる**権利のある**曲を再生。自分のアカウントの権利に依存 |
| `ENABLE_UNBLOCK` | `false` | グレー曲の代替再生。有効時、QQ や Kugou などから同名曲をマッチして再生。バージョンや音質が異なる場合あり |

## メンテナンス

- **再ログイン / アカウント切り替え** — `./relogin.sh` を実行し、再度スキャンします。
- **アップデート** — `git pull` 後に `./deploy.sh` を再実行します。ログイン状態は保持されます。

## プライバシーとセキュリティ

- 各インスタンスはユーザー自身がデプロイし、それぞれの網易雲アカウントでログインします。
- ログイン Cookie はサーバー側のボリュームにのみ存在し、フロントエンドのコード・ブラウザストレージ・URL には書き込まれません。
- 再生可否は自分のアカウントの権利に依存します。リージョン解除は既定でオフで、有効化はユーザーの判断に委ねられます。
